Working with a dataset

Usually we do not create matrices / data frames by hand (as with the previous matrix example), but instead import data from external files, e.g. from csv files.

To load a dataset, you can use the command read_csv and direct it towards the file location on your computer. For this exercise, this has already been done for you. To view the dataset, simply type in the name of the dataset “df”.

# df <- read_csv("data/alter_list.csv") 
# --> already done for you!
# Simply type in "df". Your code then looks like this:

df

After having inspected the dataset a bit, let’s only view the alters. For this we can use the $ command, which is used in R to specify a specific column.

df$alter

We now want to subset the data, only keeping the ecological nodes. For this we can use the squared brackets [] to subset and “==”, meaning “is equal to”, to indicate a subsetting condition.

PS: Don’t worry! As we do not overwrite the df, we don’t lose the rest of the dataset.

df[identity=="ecological",]

Now, let’s subset based on social alters. However, we only want to keep those social alters with daily interactions. Note how we can use the & sign for this.

df[identity=="social"&frequency=="daily",]

Let’s try subsetting the dataset for non-ecological actors (i.e. social). For this we can use the comparison operator “!=”, meaning “is different from”.

In this simple case, we could also write ==“social” as we do not have any other categories in the dataset.

df[identity!="ecological",]

Visualizing networks

After revisiting some basic R knowledge, let’s focus on using R for network analysis. Let’s start by using a single personal network.

But, what was a personal network again?

Quiz

There are different R packages for working with network data. A very versatile package is called “igraph”. To use “igraph”, we first have to transform our dataframe into an igraph object.

The code below does exactly this. Before running it, add the code snippet necessary to inspect it afterwards. You are already an R pro, so you probably don’t need a hint.

What does this output tell us? Let’s discuss together.

g <- graph_from_data_frame(df, directed=F)
#Simply type in "g" or alternatively "print(g)" and run the code.

# The graph is (U)ndirected and (N)amed. It has 14 vertices and 13 edges. It has a vertex attribute (v/c for vertex/character) called "name" and edge attributes (e/c and e/n for edge/character and edge/numberic) called "identity","frequency" and "importance".

So you see that there is already quite some information hidden in the igraph object.

To use “functions” such as the one above (graph_from_data_frame), the corresponding package has to be installed and loaded first. This can be done using R’s built-in functions install.packages(“insert_package_name”) to install and library(insert_package_name) to load the packages. In this learning environment, this has already been done for you in the background. For reference, you can take a look at the respective code here:

To plot an igraph object, we can use the “ggraph” package. Now, run the code below.

ggraph(g, layout="stress") +
geom_edge_link() + # Draw edges
geom_node_point(size=5, color="blue") + # Draw nodes
theme_graph() # Set graph theme

Congratulations, you just plotted your first network!

Network Visualization II

Let’s up our game!

We now want to color the nodes based on their “identity”.

We thus modify the code as outlined below. Do you notice the difference?

ggraph(g, layout="stress") + 
geom_edge_link() + # Draw edges / ties
geom_node_point(aes(color=identity), size=5) + # Draw nodes setting identity as node color with a fixed node size
theme_graph()
# You can try the layout "lgl" (=large graph layouts) to visualize large networks while avoiding hairballs. Not really necessary here, but looks nice :)

Let’s try to increase the node size. Any ideas on how to do that?

We now want to adjust edge width based on 1) the indicated importance of target species and 2) frequency of interaction with social alters.

ggraph(g, layout="stress") +
geom_edge_link(aes(width=importance_code)) + # Draw edges and set size based on importance
geom_edge_link(aes(width=frequency_code)) +
geom_node_point(aes(color=identity), size=5) +
theme_graph()

In a last step, we add the ID label.

ggraph(g, layout="stress") +
geom_edge_link() + # Draw edges and set size based on importance
geom_node_label(aes(color=identity,label= name)) +
theme_graph()

Network data inspection

To take a look at the node and edge data of our network, we can use V() and E(). V stands for vertices (or nodes) and E stands for edges (or links). Let’s inspect both for our network g.

V(g)
E(g)

We can use these to calculate node-level as well as edge-level characteristics of our network. For example, general network characteristics such as the degree centrality, or the mean frequency of a interactions.

E(g)$frequency
E(g)$frequency_code
mean_frequency <- mean(E(g)[identity=="social"]$frequency_code)

length(V(g)[identity=="social"])
length(V(g)[identity=="ecological"])
